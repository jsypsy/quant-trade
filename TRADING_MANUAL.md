## 1. 핵심 철학

이 시스템은 세 가지 원칙 위에서 동작한다.

### 원칙 1 — "추세를 따라간다"

> 오르는 종목은 더 오른다. 가속이 꺾이는 순간 빠져나온다.

차트의 방향성이 바뀌는 **변곡점**을 포착해서 매매한다.  
예측하지 않는다 — 시장이 먼저 움직이고, 그것을 확인한 후 진입한다.

### 원칙 2 — "작은 단위로 자주 친다"

> 한 번의 큰 베팅보다 잦은 소규모 매매로 기회를 축적한다.

- 20초마다 신호를 확인한다.
- 1회 최대 주문 금액을 제한해 단일 실수의 피해를 줄인다.
- 포지션 비중을 20%로 제한해 한 종목에 몰빵하지 않는다.

### 원칙 3 — "손실에는 자동 브레이크"

> 하루 손실이 30만 원을 넘으면 신규 매수를 전면 중단한다.

보유 중인 종목의 매도는 허용된다(포지션 정리). 새로운 진입만 막는다.

## 2. 전략 — 골든크로스 / 데드크로스

### 이동평균(MA)이란?

> **이동평균(Moving Average)**: 최근 N개 봉의 종가 평균.  
> N이 작을수록 현재 가격을 빠르게 반영하고, N이 클수록 중장기 추세를 반영한다.

예시: 20초 간격으로 가격을 수집하면  
- **MA3** = 최근 60초(1분) 평균가  
- **MA7** = 최근 140초(약 2.3분) 평균가

### 골든크로스 — 매수 신호

```
이전 봉: MA단기 ≤ MA장기
현재 봉: MA단기 > MA장기    ← 이 순간 BUY
```

```
가격
 ↑       /~~ MA단기(빠름)
 |      / /
 |-----/-/-------- MA장기(느림)
 |    / /
 |   골든크로스 발생 → BUY
```

단기 이동평균이 장기 이동평균을 **아래에서 위로 돌파**하는 순간.  
"짧은 기간의 가격 평균이 긴 기간을 앞질렀다" = 상승 모멘텀이 붙었다는 신호.

### 데드크로스 — 매도 신호

```
이전 봉: MA단기 ≥ MA장기
현재 봉: MA단기 < MA장기    ← 이 순간 SELL
```

단기 이동평균이 장기 이동평균을 **위에서 아래로 이탈**하는 순간.  
상승 모멘텀이 꺾였다 = 빠져나올 시점.

### HOLD — 아무것도 안 한다

크로스가 발생하지 않으면 매수도 매도도 하지 않는다.  
→ 신호 없이 거래하지 않는다는 규칙.

### 현재 파라미터

| 구분 | 단기창 | 장기창 | 봉 간격 | 실질 MA 기간 |
|------|--------|--------|---------|------------|
| 한국주식 | MA5 | MA20 | 20초 | 100초 vs 400초 |
| 미국주식 | MA3 | MA7 | 20초 | 60초 vs 140초 |

미국은 창을 더 좁게 잡아 신호가 더 자주 발생하도록 설계했다.

## 3. 신호 파이프라인

한 사이클(20초마다)에 다음 순서로 동작한다.

```
┌─────────────────────────────────────────────────────┐
│  TICK (20초마다)                                     │
│                                                     │
│  1. 시장 열렸는지 확인                                │
│     ├─ KR 정규장(09:00~15:30 KST) → KR 종목 처리     │
│     ├─ US 프리마켓(17:00~22:30 KST) → US 종목 처리   │
│     └─ US 정규장(22:30~05:00 KST)  → US 종목 처리   │
│                                                     │
│  2. 잔고 조회 → 총자산 / 보유종목 파악                 │
│                                                     │
│  3. 가격 수집                                        │
│     ├─ KR: KIS API (실시간 분봉)                      │
│     └─ US: Yahoo Finance (현재가 / 프리마켓가)         │
│         → 인메모리 버퍼에 쌓아 synthetic 분봉 생성       │
│                                                     │
│  4. 이동평균 계산 → 크로스 감지 → BUY / SELL / HOLD   │
│                                                     │
│  5. 리스크 가드 통과 여부 검사                         │
│                                                     │
│  6. 통과하면 KIS API로 주문 전송                       │
│                                                     │
│  7. 텔레그램 알림 (체결 시에만)                        │
│  8. trades/YYYY-MM-DD.csv 에 기록                    │
└─────────────────────────────────────────────────────┘
```

### 가격 데이터 수집 방식

**한국주식**: KIS API의 분봉 엔드포인트를 직접 호출한다. 실제 OHLCV 데이터.

> **OHLCV**: Open(시가) High(고가) Low(저가) Close(종가) Volume(거래량)의 약자. 하나의 봉(캔들)을 구성하는 5가지 정보.

**미국주식**: KIS API에 해외주식 분봉 엔드포인트가 없다.  
대신 Yahoo Finance에서 20초마다 현재가(프리마켓 중에는 프리마켓 가격)를 가져와 메모리에 쌓는다. 이 데이터로 이동평균을 계산한다.

```
버퍼: [(21:52, 131.2), (21:52:20, 131.5), (21:52:40, 131.8), ...]
         ↓ MA3 = 최근 3개 평균 = 131.5
         ↓ MA7 = 최근 7개 평균 = 131.1
         → 골든크로스 → BUY
```

신호를 생성하려면 **최소 (장기창 + 1)개**의 가격이 버퍼에 쌓여야 한다.  
MA7 기준 = 최소 8개 = 약 2.5분(시작 후 대기 시간).

## 4. 리스크 가드 — 주문 전 안전장치

신호가 BUY/SELL로 결정되더라도 주문은 다음 5단계를 통과해야 나간다.

```
신호 입력
    │
    ▼
① HOLD인가? ──Yes──→ 바로 통과 (주문 없음)
    │ No
    ▼
② 킬스위치 발동? ──Yes──→ BUY 차단 (SELL은 허용)
   (일일 손실 ≥ 30만원)
    │ No
    ▼
③ 동일 종목 미체결 주문 있음? ──Yes──→ 차단 (중복 방지)
    │ No
    ▼
④ 수량 조정
   qty = min(요청수량, 금액한도내수량, 비중한도내수량)
    │
    ▼
⑤ qty == 0? ──Yes──→ 차단
    │ No
    ▼
  주문 전송
```

### 수량 조정 계산 예시

> 총 포트폴리오 1,000만원, NVDA 현재가 $131, 환율 1,380원/달러라고 가정

- **1회 주문 한도(US)**: $500 → 최대 3주 ($131 × 3 = $393 < $500)
- **종목 비중 한도**: 포트폴리오의 20% = 200만원 / 131달러 / 1,380 ≈ 11주
- **최종 주문**: min(1, 3, 11) = **1주**

> 요청수량(qty)이 전략에서 항상 1로 설정되므로 결국 1주로 확정된다.

### 킬스위치 상세

> **킬스위치(Kill Switch)**: 손실이 임계값을 넘으면 자동으로 신규 매수를 차단하는 안전장치.

- 한도: 하루 누적 손실 **30만원**
- BUY는 차단, SELL은 허용 (손실을 더 키우지 않되, 보유 포지션은 정리 가능)
- 자정이 넘어 다음날이 되면 일일 손익이 초기화된다

## 5. 포트폴리오 배분

전체 자산을 시장별로 50:50으로 나눈다.

```
총 자산 10,000,000원
    │
    ├── 한국주식 50% (5,000,000원)
    │     ├── 삼성전자 (005930)  — 최대 비중 20%
    │     └── SK하이닉스 (000660) — 최대 비중 20%
    │
    └── 미국주식 50% (5,000,000원 상당)
          ├── NVDA (엔비디아)    — 최대 비중 20%
          └── MU (마이크론테크)   — 최대 비중 20%
```

> 두 시장은 영업시간이 겹치지 않으므로 단일 루프가 번갈아 처리한다.  
> 한국장이 열려 있으면 KR 전략만, 미국장이 열려 있으면 US 전략만 실행한다.

## 7. 주문 실행

### 주문 종류

현재 한국주식은 **지정가(LIMIT)**, 미국주식은 **시장가(MARKET)** 로 주문된다.

> **지정가(Limit Order)**: 내가 원하는 가격을 명시하는 주문. 그 가격 이하(매수) / 이상(매도)에만 체결된다.  
> **시장가(Market Order)**: 현재 시장 가격으로 즉시 체결되는 주문. 빠르지만 가격을 선택할 수 없다.

### 거래소 코드

| API 용도 | NASDAQ 코드 |
|----------|-------------|
| 시세 조회 | `NAS` (3자리) |
| 주문 전송 | `NASD` (4자리) |

> KIS API의 비일관성 — 시세 API와 주문 API에서 같은 NASDAQ을 다른 코드로 부른다.  
> 내부적으로 자동 변환 처리되어 있다.

### 모의투자(VTS) vs 실거래(PROD)

현재 설정: **모의투자(VTS)**

```
모의투자 서버: openapivts.koreainvestment.com:29443
실거래 서버:  openapi.koreainvestment.com:9443
```

모의투자는 실제 돈이 들지 않는 시뮬레이션이지만, 주문은 실제로 KIS 서버에 전송되며  
MTS(모바일 트레이딩 시스템) 앱에서 체결 내역을 확인할 수 있다.

## 10. 용어 사전

| 용어 | 설명 |
|------|------|
| **이동평균(MA, Moving Average)** | 최근 N개 봉의 종가를 평균한 값. 가격의 방향성을 부드럽게 보여준다. |
| **골든크로스(Golden Cross)** | 단기 MA가 장기 MA를 아래서 위로 돌파. 상승 전환 신호. |
| **데드크로스(Dead Cross)** | 단기 MA가 장기 MA를 위에서 아래로 이탈. 하락 전환 신호. |
| **HOLD** | 아무 행동도 하지 않음. 크로스가 발생하지 않은 상태. |
| **킬스위치(Kill Switch)** | 일정 손실 초과 시 자동으로 신규 매수를 중단하는 안전장치. |
| **모의투자(VTS)** | 실제 돈 없이 KIS 서버에 주문을 보내보는 시뮬레이션 환경. |
| **프리마켓(Pre-Market)** | 미국 정규장 개장(09:30 ET) 전 04:00~09:30 ET에 운영되는 시간외 거래 세션. |
| **OHLCV** | Open(시가)·High(고가)·Low(저가)·Close(종가)·Volume(거래량). 캔들 한 개의 구성요소. |
| **봉(Bar / Candle)** | 일정 시간 단위(예: 20초, 1분, 1일)의 가격 변동을 하나로 요약한 단위. |
| **유니버스(Universe)** | 매매 대상 종목 목록. |
| **슬리피지(Slippage)** | 주문 시 기대한 가격과 실제 체결 가격의 차이. |
| **포지션(Position)** | 현재 보유 중인 종목과 수량. 롱(매수 보유) 포지션만 다룬다. |
| **비중(Weight / Pct)** | 특정 종목이 전체 포트폴리오에서 차지하는 금액 비율. |
| **지정가(Limit Order)** | 내가 원하는 가격을 지정하는 주문. 그 가격에 도달해야 체결된다. |
| **시장가(Market Order)** | 현재 시장가로 즉시 체결되는 주문. 빠르지만 가격 통제 불가. |
| **TR_ID** | KIS API 요청 시 어떤 기능을 호출하는지 식별하는 코드. (예: VTTT1002U = 모의 해외주식 매수) |
| **Synthetic 분봉** | 실제 거래소 분봉 API 대신, 현재가를 주기적으로 수집해 직접 구성한 분봉 데이터. |
| **인메모리 버퍼** | 디스크에 저장하지 않고 프로그램 실행 중에만 메모리에 유지되는 임시 데이터. 재시작 시 초기화됨. |
