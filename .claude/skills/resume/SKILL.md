# Resume Skill — 세션 재개

이 스킬이 호출되면 마지막 세션의 컨텍스트를 복원하고 작업을 이어받습니다.

## 실행 순서

### 1. HANDOFF.md 확인
- 프로젝트 루트의 `HANDOFF.md`를 읽습니다.
- 파일이 없으면 사용자에게 알리고, 대화 기록이나 git log로 대신 맥락을 파악합니다.

### 2. 관련 메모리 로드
- memory 시스템에서 이 프로젝트와 관련된 `project`, `feedback` 타입 메모리를 확인합니다.
- 최근 메모리와 HANDOFF.md 내용이 충돌하면 더 최근 정보를 우선시합니다.

### 3. 현재 상태 검증
HANDOFF.md에 언급된 파일이나 코드가 있으면:
- 파일이 여전히 존재하는지 확인합니다.
- 언급된 함수나 심볼이 있는지 grep으로 확인합니다.
- 메모리와 현재 상태가 다르면 사용자에게 알립니다.

### 4. 재개 브리핑 출력

다음 형식으로 출력합니다:

```
## 이전 세션 요약
[완료된 작업 1-3줄 요약]

## 현재 상태
[미완료 작업 또는 막힌 지점]

## 다음 단계 제안
1. [최우선 작업]
2. [그 다음 작업]
3. [선택적 작업]

어디서부터 이어갈까요?
```

### 5. 사용자 입력 대기
브리핑 후에는 사용자가 어떤 작업을 이어갈지 선택할 때까지 기다립니다.
사용자가 방향을 바꾸거나 새 작업을 원하면 그에 맞게 전환합니다.

## HANDOFF.md가 없을 때
다음 대안 순서로 컨텍스트를 파악합니다:
1. git log --oneline -10 으로 최근 커밋 확인
2. 최근 수정된 파일 확인 (git status, git diff HEAD~1)
3. memory 시스템의 project 타입 메모리 확인
4. 파악한 내용을 바탕으로 최선의 추측을 제시하고 사용자에게 확인

## 주의사항
- 메모리의 파일 경로나 함수명은 작성 시점 기준입니다 — 현재도 존재하는지 반드시 검증하세요.
- 이전 세션의 결정을 그대로 따르지 말고, 현재 코드 상태를 우선시하세요.
- 재개 브리핑은 간결하게 — 너무 길면 오히려 방해가 됩니다.
