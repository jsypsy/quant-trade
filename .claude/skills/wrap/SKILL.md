# Wrap Skill — 세션 마무리

이 스킬이 호출되면 다음 순서로 현재 세션을 정리하고 저장합니다.

## 실행 순서

### 1. 현재 상태 파악
- 이 대화에서 다룬 작업들을 검토합니다.
- 완료된 것과 미완료된 것을 구분합니다.
- 중요한 결정, 발견한 사실, 다음에 이어야 할 것들을 정리합니다.

### 2. HANDOFF.md 작성
프로젝트 루트에 `HANDOFF.md`를 작성합니다. 파일이 이미 존재하면 덮어씁니다.

파일 구조:
```markdown
# Session Handoff
_Last updated: [날짜]_

## 이번 세션에서 완료한 것
- [완료된 작업 목록]

## 진행 중이거나 미완료
- [미완료 작업, 막힌 지점, 다음 단계]

## 핵심 결정 사항
- [이번 세션에서 내린 중요한 결정들]

## 알아두어야 할 맥락
- [다음 세션에서 바로 이해해야 할 배경 정보]

## 다음 작업 제안
- [우선순위 순으로 다음에 해야 할 일]

## 관련 파일
- [수정/생성된 주요 파일 목록]
```

### 3. 메모리 업데이트 (해당하는 경우)
다음과 같은 비자명적 사실이 있다면 memory 시스템에 저장합니다:
- 프로젝트 구조에 대한 새로운 발견
- 사용자의 선호도나 작업 방식
- 반복되는 패턴이나 피해야 할 것들

### 4. 마무리 요약 출력
사용자에게 다음을 출력합니다:
- 이번 세션 요약 (2-4줄)
- `HANDOFF.md` 저장 확인
- 다음 세션에서 `/resume`으로 이어받을 수 있음을 안내

## 주의사항
- 너무 많은 세부사항으로 HANDOFF.md를 채우지 마세요 — 핵심만 담습니다.
- 코드 스니펫은 꼭 필요한 경우에만 포함하세요 (파일 경로와 줄 번호면 충분합니다).
- HANDOFF.md는 다음 세션의 Claude가 5분 안에 파악할 수 있어야 합니다.
